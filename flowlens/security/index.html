Architecture

FlowLens is built entirely on Atlassian's Forge platform and qualifies for Runs on Atlassian. That means:

  • Zero egress. FlowLens makes no network calls outside Atlassian's infrastructure — no external servers, no third-party APIs, no CDNs, no analytics or telemetry of ours anywhere in the app.
  • All storage stays inside Atlassian. The data FlowLens keeps — issue IDs and keys, project IDs, status IDs and names, issue type IDs, and status-transition timestamps — lives in Forge-hosted storage, in your Atlassian environment, under Atlassian's own security and data-residency practices.
  • Read-only by design. FlowLens holds a single Jira scope, read:jira-work, plus app storage. It cannot create, edit, or delete anything in your Jira instance.
  • No credentials, ever. FlowLens authenticates via Forge's native OAuth mechanism. It never asks a user for a password, API token, or Personal Access Token.

What FlowLens does not do

  • No personal data collection — no names, emails, account IDs, or user identifiers of any kind
  • No reading of issue descriptions, comments, attachments, or user profiles
  • No logging of End-User Data — verified in the app's source, not just asserted
  • No remote APIs exposed for third-party integration
  • No sharing of any data, logs included, with any third party

Compliance status

FlowLens is a new listing from a small, engineer-founded vendor. We do not yet hold formal certifications (SOC 2, ISO 27001) or a completed CAIQ Lite questionnaire. What we can offer instead is an architecture with a structurally small attack surface: there is no external server to breach, no database of ours to leak, and no credential of yours we ever hold.
