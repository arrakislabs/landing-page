Getting started

  1. Install FlowLens from the Atlassian Marketplace (site admin required).
  2. Open any Jira dashboard (or create one: main nav → Dashboards → Create), click Add gadget, and search for FlowLens.
  3. In the gadget's setup form, enter your project key (e.g. ENG) and pick a time window. Save.
  4. Click Import history once. FlowLens walks your project's full changelog — you'll watch the progress live — and from then on tracks every status transition automatically. Nothing else to do, ever.

The three views

Summary

Average, median, and 85th-percentile time in each status across all issues in the window, as a bar chart and table. The 85th percentile is your "how bad does it get" number — agile coaches usually watch it over the average.

Issues

Every issue in the window with its time in each status and a total. Sort by the column you're worried about; the outliers are your stuck tickets.

Trend

Weekly average cycle time — first In Progress to Done, bucketed by completion week. Statuses are classified automatically using Jira's own status categories, so there is nothing to configure.

Configuration reference

SettingWhat it does
Project keyThe project the gadget reports on. Add multiple gadgets for multiple projects.
Time window7, 14, 30, 90, or 365 days. Issues with activity in the window are included.
Business hoursWhen on, durations count Mon–Fri 09:00–17:00 in the timezone you set (defaults to yours), instead of wall-clock time. Daylight saving is handled.

FAQ

Where does my data go?

Nowhere. FlowLens runs entirely inside Atlassian's infrastructure (Runs on Atlassian) with read-only access to work items. See the privacy policy.

What does it cost?

Free for sites with 10 or fewer users, automatically. Beyond that, a simple per-user price via Atlassian's standard billing, with a 30-day free trial.

Does the import affect my Jira?

No — the import reads your changelog at a deliberately gentle pace and writes only to the app's own storage. Your issues are never modified; FlowLens has no write permission at all.

An issue's numbers look wrong.

Check the time window (a ticket's old history may fall outside it) and whether business-hours mode is on. Still off? Email us — we ship fast.
